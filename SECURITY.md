# Security Policy

## Supported Versions

Security fixes are intended for the latest state of the default branch and the latest tagged release.

## Reporting a Vulnerability

If you discover a security issue:

1. Do not open a public issue with exploit details.
2. Contact the maintainer privately first.
3. Include affected files, reproduction steps, impact, and any suggested mitigation.

The goal is to acknowledge reports quickly, validate impact, and coordinate a fix before public disclosure when reasonable.
